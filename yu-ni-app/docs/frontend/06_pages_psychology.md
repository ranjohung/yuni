## 1. CBT思维记录表页面

### 1.1 页面定位

CBT思维记录表是认知行为疗法的核心工具，帮助用户识别和挑战自动化负面思维。当用户在模拟训练或日常对话中出现绝对化词语、灾难化词语、自我否定时，AI会建议用户填写思维记录表。

### 1.2 触发入口

| 触发方式 | 入口位置 | 触发条件 |
|---------|---------|---------|
| 对话中触发 | 聊天页面 | AI检测到认知扭曲 |
| 主动填写 | 成长页 | 用户主动点击 |
| 训练后建议 | 训练报告页 | 训练中出现负面思维 |

### 1.3 五步骤表格

```
┌────────────────────────────────┐
│ 📝 CBT思维记录表              │ 标题
├────────────────────────────────┤
│ Step 1：情境                  │
│ "发生了什么事？"              │
│ [__________________________]  │
├────────────────────────────────┤
│ Step 2：自动思维              │
│ "我当时在想什么？"            │
│ [__________________________]  │
├────────────────────────────────┤
│ Step 3：情绪                  │
│ "我感受到什么情绪？"          │
│ 😊 😢 😠 😰 😌 [其他]          │
├────────────────────────────────┤
│ Step 4：证据                  │
│ "支持这个想法的证据："        │
│ [__________________________]  │
│ "反对这个想法的证据："        │
│ [__________________________]  │
├────────────────────────────────┤
│ Step 5：替代思维              │
│ "有没有更平衡的想法？"        │
│ [__________________________]  │
├────────────────────────────────┤
│ [保存记录] [生成分析]          │
└────────────────────────────────┘
```

### 1.4 认知扭曲检测

| 扭曲类型 | 关键词 | 示例 |
|---------|--------|------|
| 绝对化 | 总是、从不、必须 | "我总是搞砸" |
| 灾难化 | 完了、糟透了、完蛋 | "我肯定会失败" |
| 过度概括 | 所有、全部、每次 | "所有人都不喜欢我" |
| 情绪推理 | 感觉就是、我觉得 | "我感觉自己很笨" |
| 自我否定 | 没用、废物、不行 | "我做不到" |

## 2. NVC可视化引导页面

### 2.1 页面定位

非暴力沟通（NVC）可视化引导帮助用户学习和实践NVC四步法：观察→感受→需要→请求。适用于冲突场景和日常沟通训练。

### 2.2 触发入口

| 触发方式 | 入口位置 | 触发条件 |
|---------|---------|---------|
| 冲突场景 | 模拟训练页面 | 进入冲突相关场景 |
| 主动学习 | 成长页 | 用户主动点击 |
| AI建议 | 聊天页面 | AI检测到沟通冲突 |

### 2.3 四步法模板

```
┌────────────────────────────────┐
│ 🗣️ NVC沟通引导                │ 标题
├────────────────────────────────┤
│ Step 1：观察                  │
│ "我观察到什么具体行为？"      │
│ [__________________________]  │
│ ❌ "你总是不理我"            │
│ ✅ "你今天没有回我消息"      │
├────────────────────────────────┤
│ Step 2：感受                  │
│ "我感受到什么情绪？"          │
│ 😊 😢 😠 😰 😌 [其他]          │
│ ❌ "你让我很失望"            │
│ ✅ "我感到失望"              │
├────────────────────────────────┤
│ Step 3：需要                  │
│ "什么需要没有得到满足？"      │
│ [被理解] [被尊重] [安全感]    │
│ [连接] [支持] [其他]          │
├────────────────────────────────┤
│ Step 4：请求                  │
│ "我可以提出什么具体请求？"    │
│ [__________________________]  │
│ ❌ "你要多关心我"            │
│ ✅ "你能每天回我一次消息吗？"│
├────────────────────────────────┤
│ [生成回复] [重新练习]          │
└────────────────────────────────┘
```

### 2.4 实时纠错规则

| 步骤 | 错误类型 | 提示信息 |
|------|---------|---------|
| 观察 | 包含评价 | "请描述具体行为，不要加入评价" |
| 感受 | 包含想法 | "请描述情绪感受，不是想法" |
| 需要 | 模糊不清 | "请明确你的具体需要" |
| 请求 | 命令语气 | "请用请求而非命令的语气" |

## 3. 情绪识别服务组件

### 3.1 功能定位

情绪识别服务通过多模态融合（文本+语音+行为）分析用户情绪状态，决定触发模式（树洞/建议/混合）。

### 3.2 成本优化策略

| 用户等级 | 文本分析 | 语音分析 | 行为分析 | 融合决策 |
|---------|---------|---------|---------|---------|
| 体验版 | 关键词匹配 | ❌ | ✅ 基础 | ✅ |
| 免费版 | 关键词匹配 | ✅ 基础 | ✅ 基础 | ✅ |
| 基础会员 | LLM轻量 | ✅ 基础 | ✅ 进阶 | ✅ |
| 标准会员 | LLM标准 | ✅ 进阶 | ✅ 进阶 | ✅ |
| 尊享会员 | LLM完整 | ✅ 完整 | ✅ 完整 | ✅ |

### 3.3 情绪状态机

```
平静 → 愉悦 → 兴奋
  │       │       │
  ▼       ▼       ▼
中性 → 焦虑 → 愤怒
  │       │       │
  ▼       ▼       ▼
低落 → 悲伤 → 绝望
```

### 6.2 数据模型

```dart
class EmotionDiary {
  final int id;
  final String content;
  final String emotion;
  final int emotionScore;
  final String insights;
  final DateTime createdAt;

  EmotionDiary({
    required this.id,
    required this.content,
    required this.emotion,
    required this.emotionScore,
    this.insights = '',
    required this.createdAt,
  });

  factory EmotionDiary.fromJson(Map<String, dynamic> json) => EmotionDiary(
    id: json['id'],
    content: json['content'],
    emotion: json['emotion'],
    emotionScore: json['emotion_score'],
    insights: json['insights'] ?? '',
    createdAt: DateTime.parse(json['created_at']),
  );

  Map<String, dynamic> toJson() => {
    'content': content,
    'emotion': emotion,
    'emotion_score': emotionScore,
  };
}
```

### 7.2 数据模型

```dart
class WeeklyReport {
  final String weekStart;
  final String weekEnd;
  final Map<String, AbilityChange> abilities;
  final TrainingStats trainingStats;
  final List<String> suggestions;
  final List<String> achievements;

  WeeklyReport({
    required this.weekStart,
    required this.weekEnd,
    required this.abilities,
    required this.trainingStats,
    required this.suggestions,
    required this.achievements,
  });

  factory WeeklyReport.fromJson(Map<String, dynamic> json) => WeeklyReport(
    weekStart: json['week_start'],
    weekEnd: json['week_end'],
    abilities: Map.from(json['abilities']).map(
      (k, v) => MapEntry(k, AbilityChange.fromJson(v)),
    ),
    trainingStats: TrainingStats.fromJson(json['training_stats']),
    suggestions: List<String>.from(json['suggestions']),
    achievements: List<String>.from(json['achievements']),
  );
}

class AbilityChange {
  final int previous;
  final int current;
  final int change;

  AbilityChange({
    required this.previous,
    required this.current,
    required this.change,
  });

  factory AbilityChange.fromJson(Map<String, dynamic> json) => AbilityChange(
    previous: json['previous'],
    current: json['current'],
    change: json['change'],
  );
}
```
